new in version 3.5.0
********************

each slide of the collection has now a sequence number. the sequence number allows for custom sort order in the collection window. in a future version, slides can optionally be played in the order given by the sequence number.

sequence number, display frequency, or display time can be changed via drag & drop in the collection window while the respective sort key is selected.

multi-monitor support: the new slide show -> monitor menu item selects the monitor on which the slide show is displayed. the monitor can also selected on the command line, using the -monitor token followed by the monitor number (1, 2, 3, ...). like the file parameters, the monitor parameter can be given multiple times, and will affect the files listed after it. the following example will open two slide shows on two monitors:

karsten.exe -monitor 1 -maximize demo1.kbs -monitor 2 -maximize demo2.kbs

bugfix: wrong sort order for frequency and time parameters

workaround (bug 1623388): karsten uses 100% CPU in full-screen slide show mode (while remaining fully responsive). the reason for this issue eludes me. as a workaround, some menu and toolbar items are not updated while the collection window is not active. click on the slide show window or any window of another application while watching the slide show if you care about minimizing CPU load. (programmers: see comment to TCollectionForm.AViewStyleUpdate in sammelfen.pas).

neu in version 3.2.10
*********************

verwendet den windows-verknüpfungsmechanismus, um beim öffnen einer sammlungsdatei verschobene oder umbenannte bilddateien zu finden und die pfadangaben zu aktualisieren.
windows 2000, xp und neuer führen auf NTFS-laufwerken buch ueber verschobene und umbenannte dateien, die suche ist in diesem fall sehr effizient. in anderen windows-versionen und auf FAT/FAT32-laufwerken wird nur eine heuristische suche anhand von datum und grösse der datei ausgeführt. die erfolgschancen hängen davon ab, wie weit die datei von ihrem ursprünglichen ort entfernt wurde.

verwendet die moderne directx-technologie, um multimedia-dateien abzuspielen. dadurch können alle medienformate, die der windows-mediaplayer abspielen kann, abgespielt werden. für einige neuere formate werden nur noch directx-codecs angeboten, die karsten bisher nicht abspielen konnte.
für die medienwiedergabe muss eine einigermassen aktuelle directx-version installiert sein (gratis erhältlich von der microsoft-website), empfohlen wird mindestens version 8.1, möglicherweise funktioniert karsten aber auch mit einigen früheren versionen.
